Chaque commande a une description (affiché sur le bouton de la page principale), et une commande actuel à exécuter. Cette commande peut contenir des opérateurs de l'interpréteur de commande comme | et ; pour l'exécution de commandes multiples et de transferts de données. Il peut aussi contenir des paramètres comme $foo, qui sont remplacer par une saisie de l'usager quand la commande est exécutée.

Ces paramètres peuvent être entré dans la table au bas de la page. Pour chaque pramètre vous devez entrer :

Nom
Un code unique pour ce paramètre. Si le nom est foo, bien $foo sera remplacé par la valeur de ce paramètre quand la commande est exécutée.

Description
La description après le paramètre sur la page principale.

Type
Cette option contrôle comment le paramètre est entré. Les options valides sont :
  • Texte
    Une entrée libre de texte.
  • Usager
    Un nom d'usager sur votre système.
  • Numéro d'Usager
    Le numéro d'un usager de votre système.
  • Groupe
    Un nom de groupe de votre système.
  • Numéro de Groupe
    Le numéro d'un groupe de votre système.
  • Fichuer
    Le chemin complet d'un fichier.
  • Répertoire
    Le chemin complet d'un répertoire.
  • Option
    Une sasie Oui/Non qui va définir le paramètre à n'importe quoi qui est dans le champs après l'entrée de ce type si Oui est choisit.
  • Mot de Passe
    Une saisie de texte libre, mais où le contenu va être remplacé par des * à l'affichage.

Paramètre entre "" ?
Si Oui, le paramètre va etre mit entre "", permettant à l'usager d'entrer des valeurs contenant des espaces.
