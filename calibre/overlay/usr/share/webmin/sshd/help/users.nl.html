Authenticatie
Deze pagina is om authenticatie gerelateerde opties te configureren voor uw SSH server. Sommige minder duidelijk opties zijn :
Authenticatie met wachtwoord toestaan
Indien deze optie is ingesteld op Nee, moet bij de gebruikers hun .ssh directory worden ingesteld met een publieke sleutel voordat zij kunnen inloggen. Indien dit op Ja staat, zal de gebruiker kunnen inloggen met een wachtwoord voor authenticatie indien de validatie niet kan worden gedaan met een publieke sleutel.

RSA authenticatie toestaan?
Indien dit is ingesteld op Nee, moeten gebruikers altijd hun wachtwoord invullen ook al is hun publieke sleutel ingesteld..

Check permissies op sleutel files?
Indien dit is ingesteld op Ja, zal de SSH server iedere gebruikers .ssh directory en iedere hoofd directory controleren om er zeker van te zijn dat zij niet de groep schrijf rechten hebben, en de .ssh/identity file om er zeker van te zijn dat die niet leesbaar is door andere personen. Indien dit is ingesteld op Nee, zal er geen controle worden uitgevoerd op file permissies.
