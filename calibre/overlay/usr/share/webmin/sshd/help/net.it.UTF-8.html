Reti
Questa pagina contiene le opzioni del server SSH sulle reti. Alcune delle opzioni meno ovvie sono:
Timeout inattività
Se non vi sono dati inviati o ricevuti dal server per questo periodo la connessione viene terminata.

Disconnetti se il client cade?
Se impostata su Sì, il server SSH controlla periodicamente se il client è ancora attivo. Se non risponde (per errori di rete o per caduta del client) il server chiude la connessione.

Consenti redirezione TCP?
Se questa opzione è attiva, gli utenti possono incanalare le connessioni TCP dalle loro macchine a macchine presenti nella rete del server, e inoltre abilita connessioni inverse dal server a macchine nella rete del client.
