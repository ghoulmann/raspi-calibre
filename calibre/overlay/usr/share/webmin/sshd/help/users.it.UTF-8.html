Autenticazione
Questa pagina contiene opzioni relative all' autenticazione sul server SSH. Alcune delle opzioni meno ovvie sono :
Consenti autenticazione via password?
Se impostata su No, gli utenti hanno bisogno delle loro directory .ssh contenenti le proprie chiavi pubbliche prima di poter effettuare un login. Se impostata su Sì, gli utenti sono autorizzati ad immetter la propria password per autenticarsi se non è possibile farlo tramite chiave pubblica.

Consenti autenticazione RSA (SSH 1)?
Se impostata su No, gli utenti devono sempre inserire la loro password anche se è disponibile una chiave pubblica funzionante.

Controlla permessi sui file della chiave?
Se impostata su Si, il server SSH controlla ogni directory utente .ssh e ogni sotto directoty per assicurarsi che non sia attivo il permesso di scrittura per il gruppo, e il file .ssh/identity non sia leggibile da chiunque. Se impostata su No, nessun controllo verrà effettuato
