Действие
Описывает, какие действия необходимо предпринять.

Допустимые значения для данного поля :

Перезапуск
Процесс будет рестартован при его завершении (пример: getty).

Ожидание
Процесс будет запущен единожды при входе в соответствующий уровень выполнения и init будет ждать его завершения.

Единожды на уровне
Процесс будет выполнен единожды при входе в соответствующий уровень выполнения.

При старте
Процесс будет выполнен при загрузки системы. Содержимое поля уровней выполнения игнорируется.

При старте, ждать окончания
Процесс будет выполнен при загрузки системы, init будет ждать его завершения (например /etc/rc).Содержимое поля уровней выполнения игнорируется.

Ничего не делать
Это значит ничего не делать.

При нажождении в A, B, C
Процесс имеющий уровень выполнения ondemand будет выполнятся, когда вызывается указанный уровень выполнения ondemand. Однако изменения уровня выполнения не произойдет (`a', `b' и `c' - уровни выполнения ondemand).

На уровне выполнения по умолчанию
Уровень initdefault является уровнем выполнения в который система входит после загрузки. Если не существует, init попросит ввести этот уровень с консоли после загрузки. Содержимое поля процесса игнорируется.

Во время старта системы
Процесс будет выполнен в течении загрузки системы. Он будет выполнен до любого При старте или При старте, ждать окончания процессов. Содержимое поля уровней выполнения игнорируется.

Отключение питания
Процесс будет выполнен при отключение питания. Обычно init узнает об этом от программы управляющей ИБП (UPS), который подключен к компъютеру. Init будет ждать окончания данного процесса перед тем, как продолжить свое дальнейшее выполнение.

Отключение питания без ожидания
Аналогично, как Отключение питания за исключением того, что init не ждет завершения процесса.

Восстановление питания
Данный процесс будет выполнен, как только init узнает о том, что питание восстановлено.

Сбой питания
Данный процесс будет выполнен, когда init узнает о том, что ресурс аккумуляторных баттарей внешнего ИБП на исходе и питание скоро исчезнет (предпологается, что внешний ИБП и программа его мониторинга способны обнаружить данную ситуацию).

Ctrl-Alt-Del
Данный процесс будет выполнен, когда init получит сигнал SIGINT. Это означает, что некто нажал комбинацию клавиш CTRL-ALT-DEL на консоли. Обычно на Desktop системах данный процесс осуществляет некий вид завершения работы (например переход в одноползовательский режим или перезагрузка машины).

Специальная комбинация клавиш
Данный процесс будет выполнен, когда init получит сигнал от обработчика клавиатуры о том, что была нажата специальная комбинация клавиш на консоли.
